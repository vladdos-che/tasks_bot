# Руководство по развертыванию на Ubuntu Server

В данном руководстве описан процесс развертывания проекта (Бэкенд, Бот и Фронтенд) на сервере под управлением Ubuntu с использованием **Docker** и **Docker Compose**.

## Шаг 1: Подготовка сервера

Подключитесь к вашему серверу по SSH:
```bash
ssh user@your_server_ip
```

Обновите пакеты:
```bash
sudo apt update && sudo apt upgrade -y
```

## Шаг 2: Установка Docker и Docker Compose

Если Docker еще не установлен, установите его с помощью официального скрипта:
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

Установите плагин Docker Compose:
```bash
sudo apt install docker-compose-plugin -y
```

*(Опционально)* Добавьте вашего пользователя в группу docker, чтобы запускать команды без `sudo`:
```bash
sudo usermod -aG docker $USER
newgrp docker
```

## Шаг 3: Копирование файлов проекта на сервер

Склонируйте ваш репозиторий с проектом (если используете git):
```bash
git clone https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
cd ВАШ_РЕПОЗИТОРИЙ
```

*Либо* скопируйте файлы с помощью `scp` / `rsync`.

## Шаг 4: Настройка переменных окружения и базы данных

1. Скопируйте файл конфигурации:
```bash
cp .env.example .env
```
2. Откройте `.env` и настройте токены:
```bash
nano .env
```
Обязательно укажите `BOT_TOKEN` и другие секреты.

3. Создайте пустой файл базы данных (чтобы Docker примонтировал его как файл, а не как папку):
```bash
touch schedule.sqlite3
```

## Шаг 5: Запуск проекта

Убедитесь, что порт 2525 свободен.

Соберите и запустите контейнеры в фоновом режиме:
```bash
docker compose up -d --build
```

**Проверка статуса:**
```bash
docker compose ps
```
Вы должны увидеть три работающих контейнера: `local_ai_tasks_api`, `local_ai_tasks_bot`, и `local_ai_tasks_frontend`.

После этого сайт, бот и бэкенд запустятся внутри Docker.
- Фронтенд будет доступен на порту **2525**.
- Бэкенд (API) будет напрямую доступен на порту **5252**.

## Шаг 6. Настройка домена в HestiaCP
Теперь нужно сделать так, чтобы ваш красивый домен вел на этот Docker-контейнер.
1. Зайдите в панель Hestia.
2. Перейдите в раздел **Web** и нажмите **Add Web Domain**.
3. Введите ваше доменное имя.
4. После создания домена нажмите на иконку карандаша (Edit) рядом с ним.
5. Найдите раздел **Advanced Options** -> **Proxy Support (Nginx)**.
6. Выберите прокси-шаблон (Proxy Template) — там обычно есть шаблон **`docker`** или **`proxy`**.
7. В поле **Proxy Port** укажите `2525`.
8. Нажмите **Save**. Не забудьте также включить SSL сертификат (Let's Encrypt) в настройках домена.

Всё! HestiaCP сама примет запрос по вашему домену (со стандартного 80/443 порта) и прозрачно перенаправит его в наш Docker-контейнер на порт 2525.

## Дополнительно: Остановка и Обновление

**Остановка:**
```bash
docker compose down
```

**Обновление проекта (после копирования новых файлов):**
```bash
docker compose up -d --build
```
